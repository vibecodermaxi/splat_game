    - Test 2: RoundHistory.read returns empty array when file contains invalid JSON
    - Test 3: RoundHistory.push adds entry and persists to file
    - Test 4: RoundHistory.push keeps only the last 5 entries (oldest dropped)
    - Test 5: After push, reading from a new RoundHistory instance returns the same data (persistence verified)
    - Test 6: RoundHistory.push writes atomically (temp file then rename)
  </behavior>
  <action>
    Create `oracle/src/history.ts`:
    - Export `RoundHistory` class:
      - Constructor: accepts `filePath: string` (default: `path.join(process.cwd(), 'round_history.json')`)
      - `read(): Promise<RoundHistoryEntry[]>`:
        - Try reading filePath with fs.readFile
        - Try JSON.parse — if it fails (corrupt file), log warning and return []
        - If file not found (ENOENT), return []
        - Return parsed array
      - `push(entry: RoundHistoryEntry): Promise<void>`:
        - Read current history
        - Append entry
        - Keep only last 5: `history.slice(-5)`
        - Write atomically:
          1. Write to `filePath + '.tmp'` with fs.writeFile
          2. Rename `.tmp` to final path with fs.rename (atomic on Linux/macOS)
        - Log: { event: 'history_updated', entries: history.length }
      - `reconstruct(chain: ChainClient, seasonNumber: number, currentPixel: number): Promise<void>`:
        - For pixels max(0, currentPixel-5) to currentPixel-1:
          - Fetch PixelState from chain
          - If Resolved, create RoundHistoryEntry (note: reasoning will be empty string since it's not stored on-chain)
          - Push to local history array
        - Write reconstructed history to file
        - This handles Railway container replacement where round_history.json is lost

    Create `oracle/src/__tests__/history.test.ts`:
    - Use a temp directory for test files (create in beforeEach, clean in afterEach)
    - Test all 6 behaviors above
    - For atomicity test: verify `.tmp` file does not exist after push completes
  </action>
