1. **useBetHistory hook** (`app/src/hooks/useBetHistory.ts`):
   - Export interface `BetHistoryEntry`:
     ```
     pixelIndex: number
     colorIndex: number
     amount: number        // lamports
     claimed: boolean
     seasonNumber: number
     betPDA: string         // base58 PublicKey string
     pixelPDA: string       // base58 PublicKey string for claim accounts
     statsPDA: string       // base58 PublicKey string for claim accounts
     ```
   - Export interface `PlayerStats`:
     ```
     totalBets: number
     totalVolume: number     // lamports
     correctPredictions: number
     hitRate: string         // e.g. "42.5"
     ```
   - Hook signature: `useBetHistory()` returns `{ bets: BetHistoryEntry[], stats: PlayerStats | null, loading: boolean, error: string | null, refresh: () => void }`
   - Implementation:
     a. Use `useAnchorProgram()` and `useWallet()` to get program and wallet.
     b. On call to `refresh()` (or initial mount if wallet connected), fetch all BetAccounts for the player:
        ```
        const betAccounts = await (program.account as any)["betAccount"].all([
          { memcmp: { offset: 8, bytes: wallet.publicKey.toBase58() } },
        ]);
        ```
     c. For each bet account, also resolve the winning color by checking the corresponding PixelState. Fetch all unique pixel PDAs via `fetchMultiple` to determine which bets are wins vs losses.
     d. Map to `BetHistoryEntry[]`, computing `pixelPDA` and `statsPDA` for each entry (needed for claim transactions).
     e. Fetch PlayerSeasonStats PDA to populate `PlayerStats`. Derive via `deriveStatsPDA(PROGRAM_ID, seasonNumber, wallet.publicKey)`. If account doesn't exist, return null stats.
     f. Compute `hitRate` as `(correctPredictions / totalBets * 100).toFixed(1)`.
     g. Do NOT fetch on every render — only when `refresh()` is called or on initial mount when wallet is connected. Store results in local useState.

2. **useClaimWinnings hook** (`app/src/hooks/useClaimWinnings.ts`):
   - Hook signature: `useClaimWinnings()` returns `{ claimSingle: (bet: BetHistoryEntry) => Promise<boolean>, claimAll: (bets: BetHistoryEntry[]) => Promise<number>, isClaiming: boolean, claimProgress: string | null, error: string | null, clearError: () => void }`
   - `claimSingle(bet)`: Submits a single `claim_winnings` instruction using the bet's pixelPDA, betPDA, and statsPDA. Returns true on success, false on failure. Sets error on failure.
   - `claimAll(bets)`: Loops sequentially (`for...of`, NOT Promise.all) through claimable bets. Updates `claimProgress` string (e.g., "Claiming 2 of 5..."). Returns count of successful claims. IMPORTANT: sequential execution required because all claims write the same PlayerSeasonStats PDA — parallel submission causes "account already in use" errors.
   - Transaction pattern (same as usePlaceBet):
     ```
     await (program.methods as any)
       .claimWinnings()
       .accounts({
         pixelState: new PublicKey(bet.pixelPDA),
         betAccount: new PublicKey(bet.betPDA),
         playerSeasonStats: new PublicKey(bet.statsPDA),
         player: wallet.publicKey,
         systemProgram: SystemProgram.programId,
       })
       .rpc({ commitment: "confirmed" });
     ```
   - Double-submit protection: ignore calls while `isClaiming` is true.
   - On successful claim, the caller (ClaimButton) should trigger `refresh()` from useBetHistory.
