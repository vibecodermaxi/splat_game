   - Export `startHealthServer(port?: number)` that creates an HTTP server
   - Port defaults to `parseInt(process.env.PORT || "3000", 10)` (Railway sets PORT)
   - GET /health returns 200 with JSON: `{ status: "ok", uptime: process.uptime(), timestamp: new Date().toISOString() }`

   - Log `{ event: "health_server_started", port }` using the logger from ./logger
   - Keep it simple: ~30 lines, no Express dependency needed

2. Modify oracle/src/index.ts:
   - Add `import { startHealthServer } from "./health"` at the top imports
   - Call `startHealthServer()` BEFORE the recovery check (right after module initialization, around line 59, after the logger.info oracle_starting event)
   - This ensures the health endpoint is responsive even during recovery

3. Create oracle/Dockerfile:
   ```dockerfile
   FROM node:20-slim
   WORKDIR /app
   COPY package.json package-lock.json ./
   RUN npm ci --omit=dev
   COPY tsconfig.json ./
   COPY src/ ./src/
   RUN npx tsc
   ENV PORT=3000
   EXPOSE 3000
   CMD ["node", "dist/index.js"]
   ```
   NOTE: The oracle/src/idl.json is gitignored but MUST exist for `tsc` to succeed. The Dockerfile copies src/ which includes whatever idl.json is present locally. Before building the Docker image or pushing to Railway, the user must have run `scripts/copy-idl.sh` so that oracle/src/idl.json exists. Add a comment in the Dockerfile noting this requirement.

4. Create oracle/.dockerignore:
   ```

   dist

   round_history.json
   ```
