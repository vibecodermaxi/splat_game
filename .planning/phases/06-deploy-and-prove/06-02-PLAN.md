  <action>
Create scripts/e2e-smoke-test.ts — a semi-automated E2E test that runs the full game loop on devnet.

Requirements: Loads env from oracle/.env (same as devnet-setup.ts). Uses the oracle keypair as the "player" for testing (it has SOL from airdrop). Does NOT require the oracle service to be running — it drives the round lifecycle directly via on-chain instructions.

Script flow:

1. **Setup phase:**
   - Load env vars from oracle/.env via dotenv
   - Parse SOLANA_RPC_URL, ORACLE_KEYPAIR, PROGRAM_ID from env
   - Create Connection, Wallet, AnchorProvider, Program (same pattern as devnet-setup.ts)
   - Load IDL from oracle/src/idl.json
   - Print "E2E Smoke Test Starting..."
   - Record initial SOL balance of the test wallet: `connection.getBalance(wallet.publicKey)`

2. **Check season state:**
   - Derive season PDA for season 1
   - Fetch SeasonState: `program.account.seasonState.fetch(seasonPda)`
   - Read `currentPixelIndex` — this is the pixel we'll bet on
   - Print "Season 1, pixel index: {N}"

3. **Open a round (if needed):**
   - Derive PixelState PDA for (season=1, pixelIndex=currentPixelIndex)
   - Check if pixel already exists: `connection.getAccountInfo(pixelPda)`
   - If not exists, call `open_round` instruction:
     ```typescript
     const pixelBuf = Buffer.alloc(2);
     pixelBuf.writeUInt16LE(currentPixelIndex, 0);
     await program.methods
       .openRound(1, currentPixelIndex)  // season_number, pixel_index as instruction args
       .accounts({
         pixelState: pixelPda,
         seasonState: seasonPda,
         config: configPda,
         oracle: wallet.publicKey,
         systemProgram: SystemProgram.programId,
       })
       .rpc();
     ```
   - If exists, check status. If "resolved", the season may have advanced — re-fetch season state and retry with new currentPixelIndex
   - Print "Round opened for pixel {N}" or "Round already open for pixel {N}"

4. **Place a bet:**
   - Choose color index 0 (Red) as the test bet
   - Bet amount: 0.05 SOL (50_000_000 lamports)
   - Derive BetAccount PDA: seeds = ["bet", season_u16_le, pixel_u16_le, player_pubkey]
   - Derive PlayerSeasonStats PDA: seeds = ["player_stats", season_u16_le, player_pubkey]
   - Call `place_bet` instruction:
     ```typescript
     await program.methods
       .placeBet(1, currentPixelIndex, 0, new anchor.BN(50_000_000))  // season, pixel, color, amount
       .accounts({
         betAccount: betPda,
         pixelState: pixelPda,
         seasonState: seasonPda,
         playerSeasonStats: playerStatsPda,
         player: wallet.publicKey,
         systemProgram: SystemProgram.programId,
       })
       .rpc();
     ```
   - Verify BetAccount exists: `connection.getAccountInfo(betPda)` is not null
   - Print "Bet placed: 0.05 SOL on Red for pixel {N}"

5. **Lock the round:**
   - Wait for the betting window to close. Since we control the round, we can immediately lock it for testing purposes.
   - IMPORTANT: The lock instruction checks that `Clock::get().unix_timestamp >= pixel_state.opened_at + BETTING_WINDOW_SECONDS`. For devnet with 5-min rounds (300s total, 180s betting window), we need to wait. Use `await sleep(5000)` for a small buffer, then try locking. If it fails with "Round cannot be locked yet", poll every 5 seconds up to 200 seconds.
   - Call `lock_round`:
     ```typescript
     await program.methods
       .lockRound(1, currentPixelIndex)  // season_number, pixel_index as instruction args
       .accounts({
         pixelState: pixelPda,
         config: configPda,
       })
       .rpc();
     ```
   - Print "Round locked for pixel {N}"

6. **Resolve the round:**
   - Wait the lock window (2 minutes = 120 seconds). Poll every 10 seconds.
   - Choose a winning color: use color index 0 (Red) so our bet wins
   - Call `resolve_round`:
     ```typescript
     const treasuryWallet = new PublicKey("6vTe3xRjB4Hv4fN4WQ5xtcF21Ed12DFPoNwHJTZDUg5v");
     const jackpotWallet = new PublicKey("HrfnbCNRzvekRkdUJGzvmEu478F43uk7weReNDPqv2TB");
     await program.methods
       .resolveRound(1, currentPixelIndex, 0, 2, 3, "Smoke test resolution")
       // args: season, pixel, colorIndex, shade, warmth, reasoning
       .accounts({
         pixelState: pixelPda,
         seasonState: seasonPda,
         config: configPda,
         oracle: wallet.publicKey,
         treasury: treasuryWallet,
         jackpot: jackpotWallet,
         systemProgram: SystemProgram.programId,
       })
       .rpc();
     ```
   - Print "Round resolved: Red wins for pixel {N}"

7. **Claim winnings:**
   - Call `claim_winnings`:
     ```typescript
     await program.methods
       .claimWinnings(1, currentPixelIndex)  // season_number, pixel_index
       .accounts({
         betAccount: betPda,
         pixelState: pixelPda,
         playerSeasonStats: playerStatsPda,
         player: wallet.publicKey,
         systemProgram: SystemProgram.programId,
       })
       .rpc();
     ```
   - Print "Winnings claimed for pixel {N}"

8. **Verify final state:**
   - Check final SOL balance: `connection.getBalance(wallet.publicKey)`
   - Compare to initial balance — should be close to initial (minus tx fees, since we were the only bettor and won 95% back)
   - Fetch the BetAccount and verify `claimed` field is true
   - Fetch the PixelState and verify status is "resolved"
   - Print balance summary: "Initial: {X} SOL, Final: {Y} SOL, Fees: {diff} SOL"
   - Print "PASS" if all checks succeed, "FAIL: {reason}" if any check fails
   - Exit with code 0 on PASS, 1 on FAIL

Helper functions needed:
- `sleep(ms: number): Promise<void>` — simple setTimeout wrapper
- `derivePDA(seeds: Buffer[], programId: PublicKey): PublicKey` — or just use inline findProgramAddressSync
- `u16ToBuffer(n: number): Buffer` — same pattern as devnet-setup.ts

IMPORTANT constraints:
- Use `anchor.BN` for lamport amounts (not plain numbers)
- Use `{ commitment: "confirmed" }` on provider for faster confirmations on devnet
- The oracle keypair from env IS the oracle authority (it was used to deploy and init the program)
- The script uses the oracle keypair as both oracle AND player — this is fine for smoke testing
- open_round and lock_round take season_number and pixel_index as instruction ARGUMENTS (not just PDA seeds) — this is a key decision from Phase 1 to avoid Anchor 0.32 self-referential PDA validation
- Add a timeout: if any step takes more than 5 minutes, print FAIL with timeout reason and exit 1
- Add a `--help` flag that prints usage
