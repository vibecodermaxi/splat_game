**place_bet instruction:**

Accounts struct `PlaceBet`:
- `season_state: Account<'info, SeasonState>` with `mut`, appropriate seeds/bump, `constraint = season_state.status == SeasonStatus::Active @ PixelPredictError::SeasonNotActive`
- `pixel_state: Account<'info, PixelState>` with `mut`, seeds: `[b"pixel", season_state.season_number.to_le_bytes(), pixel_index.to_le_bytes()]`, bump = pixel_state.bump
  - Constraint: `pixel_state.status == RoundStatus::Open @ PixelPredictError::RoundNotOpen`
- `bet_account: Account<'info, BetAccount>` with `init_if_needed`, `payer = player`, `space = 8 + BetAccount::INIT_SPACE`, seeds: `[b"bet", season_state.season_number.to_le_bytes(), pixel_index.to_le_bytes(), player.key()]`, bump
  - Constraint: `!bet_account.claimed @ PixelPredictError::AlreadyClaimed` (re-init guard)
- `player_season_stats: Account<'info, PlayerSeasonStats>` with `init_if_needed`, `payer = player`, `space = 8 + PlayerSeasonStats::INIT_SPACE`, seeds: `[b"stats", season_state.season_number.to_le_bytes(), player.key()]`, bump
- `player: Signer<'info>` (also payer)
- `system_program: Program<'info, System>`

Arguments: `pixel_index: u16, color: u8, amount: u64`

Handler logic:
1. Validate color < NUM_COLORS (error: InvalidColor)
2. Validate amount >= MIN_BET (error: BetTooSmall)
3. Lockout check: `Clock::get()?.unix_timestamp < pixel_state.opened_at + BETTING_WINDOW_SECONDS` (error: BettingLocked)
4. If bet_account.amount > 0 (existing bet): validate `bet_account.color == color` (error: ColorMismatch)
5. Validate `bet_account.amount.checked_add(amount) <= MAX_BET_PER_COLOR` (error: BetTooLarge)
6. Transfer SOL from player to pixel_state PDA via `system_program::transfer` CPI
7. Update bet_account: set player, season_number, pixel_index, color, `amount += amount` (use checked_add), claimed=false, bump
   - Handle init_if_needed: if first bet (amount was 0), set all fields. If increase, only add to amount.
8. Update pixel_state: `color_pools[color as usize] += amount`, `total_pool += amount` (both checked_add)
9. Update player_season_stats:
   - If first time (total_bets == 0): set player pubkey, season_number, bump. Also increment `season_state.unique_wallets`
   - `total_bets += 1`, `total_volume += amount`, `colors_bet[color as usize] += 1`
10. Update season_state: `total_volume += amount`, `total_bets += 1`

**SC-17 Future pixel betting:**
The PixelState must exist (created by open_round) for the player to bet. Plan 01-02 modified open_round so that when the oracle opens pixel N, it also pre-opens pixel N+1 (with Open status). This means the PixelState for the next pixel already exists. The constraint `pixel_state.status == RoundStatus::Open` handles this -- the pre-opened next pixel is in Open status. No special logic needed in place_bet beyond the existing status check. If a future pixel's PixelState doesn't exist, the transaction fails (account not found).

For the SC-17 test: after opening pixel 0 (which also pre-opens pixel 1), place a bet on pixel 1. This should succeed because pixel 1's PixelState exists with Open status.

**Important: init_if_needed behavior.**
When using `init_if_needed` on BetAccount, Anchor will initialize the account if it doesn't exist (paying rent from player). On subsequent calls, it skips init but still validates constraints. The handler must distinguish first-bet vs increase:
- Check `bet_account.amount == 0` as proxy for "newly initialized" (since first call sets amount to 0 during init, then handler adds to it). Actually, `init_if_needed` initializes all fields to their default (0/false/etc), so on first call bet_account.amount will be 0 and bet_account.color will be 0. This means we must set color on first call and check it on subsequent calls.
- Better approach: check if `bet_account.player == Pubkey::default()` as the "is new" flag. On first bet, player is default (zeroed). Set it to the actual player key. On subsequent bets, it's already set.

4. `pixel_state.locked_at = Some(clock.unix_timestamp)`
