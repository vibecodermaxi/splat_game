**If ORAO crate builds successfully -- implement full on-chain verification:**

Create `resolve_round_vrf` as a SEPARATE instruction from resolve_round:

Accounts struct `ResolveRoundVrf`:
- `season_state: Account<'info, SeasonState>` with `mut`, seeds, bump, `constraint = season_state.status == SeasonStatus::Active`
- `pixel_state: Account<'info, PixelState>` with `mut`, seeds, bump
  - Constraint: `pixel_state.status == RoundStatus::Open || pixel_state.status == RoundStatus::Locked`
  - Constraint: `pixel_state.winning_color.is_none() @ PixelPredictError::RoundAlreadyResolved`
- `config: Account<'info, ConfigAccount>` with seeds, bump, `has_one = oracle`
- `oracle: Signer<'info>`
- `/// CHECK: Treasury wallet` with `#[account(mut, address = TREASURY_WALLET)]`
- `/// CHECK: Jackpot wallet` with `#[account(mut, address = JACKPOT_WALLET)]`
- `/// CHECK: ORAO VRF randomness account -- verified by owner check and deserialization` with `#[account(owner = orao_solana_vrf::program_id())]` (or the known ORAO program ID constant)
- `system_program: Program<'info, System>`

Handler:
1. Deserialize the ORAO randomness account using `orao_solana_vrf::state::Randomness::try_deserialize()` or manual byte reading
2. Verify the randomness request is fulfilled (status check -- ORAO uses a `fulfilled` field or the randomness bytes are non-zero)
3. Extract `winning_color = randomness_bytes[0] % 16`
4. Set `pixel_state.vrf_resolved = true`
5. Set `pixel_state.shade = Some(50)` and `pixel_state.warmth = Some(50)` (spec: VRF resolution uses shade 50, warmth 50)
6. Set `pixel_state.winning_color = Some(winning_color)`
7. Set `pixel_state.status = RoundStatus::Resolved`, `pixel_state.resolved_at = Some(Clock::get()?.unix_timestamp)`
8. Perform rake transfer (same as resolve_round): 3% to treasury, 2% to jackpot via direct lamport mutation
9. Handle zero-winner branch: if `color_pools[winning_color] == 0`, transfer net pool to treasury
10. Season completion: increment `current_pixel_index`, check if season is complete

Extract the shared rake/zero-winner/season-completion logic into a helper function in a shared module (e.g., `src/resolution_helpers.rs` or inline in the instruction) to avoid duplicating the code from resolve_round.
