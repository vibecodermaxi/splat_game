**payout.rs** -- pure function, testable in isolation:

```rust
use anchor_lang::prelude::*;
use crate::errors::PixelPredictError;

/// Calculates a winner's payout using u128 intermediates and multiply-before-divide.
/// net_pool = total_pool * 95 / 100 (rake already transferred separately)
/// payout = bet_amount * net_pool / winning_color_pool
pub fn calculate_winner_payout(
    bet_amount: u64,
    winning_color_pool: u64,
    total_pool: u64,
) -> Result<u64> {
    let net_pool = (total_pool as u128)
        .checked_mul(95)
        .ok_or(error!(PixelPredictError::ArithmeticOverflow))?
        .checked_div(100)
        .ok_or(error!(PixelPredictError::ArithmeticOverflow))?;

    let payout = (bet_amount as u128)
        .checked_mul(net_pool)
        .ok_or(error!(PixelPredictError::ArithmeticOverflow))?
        .checked_div(winning_color_pool as u128)
        .ok_or(error!(PixelPredictError::ArithmeticOverflow))?;

    u64::try_from(payout).map_err(|_| error!(PixelPredictError::ArithmeticOverflow))
}
```

**resolve_round instruction:**

Accounts struct `ResolveRound`:

- `pixel_state: Account<'info, PixelState>` with `mut`, seeds, bump

  - Constraint: `pixel_state.winning_color.is_none() @ PixelPredictError::RoundAlreadyResolved`
- `config: Account<'info, ConfigAccount>` with seeds, bump, `has_one = oracle`
- `oracle: Signer<'info>`
- `/// CHECK: Treasury wallet -- verified against program constant` with `#[account(mut, address = TREASURY_WALLET @ PixelPredictError::InvalidTreasury)]` -- add InvalidTreasury error
- `/// CHECK: Jackpot wallet -- verified against program constant` with `#[account(mut, address = JACKPOT_WALLET @ PixelPredictError::InvalidJackpot)]` -- add InvalidJackpot error
- `system_program: Program<'info, System>`

Arguments: `winning_color: u8, shade: u8, warmth: u8, vrf_resolved: bool`

Note: resolve_round does NOT accept prompt_data per user decision. Arweave txId will be stored separately (Plan 05 adds an update_arweave_txid instruction or it's set in resolve_round as an optional arg).

Handler logic:
1. Validate `winning_color < NUM_COLORS`, `shade <= 100`, `warmth <= 100`
2. Set pixel_state fields: winning_color, shade, warmth, vrf_resolved, status=Resolved, resolved_at=Clock
3. Calculate rake:
   ```rust
   let total_pool = pixel_state.total_pool;
   if total_pool > 0 {
       let treasury_cut = (total_pool as u128)
           .checked_mul(TREASURY_BPS as u128).unwrap()
           .checked_div(10_000).unwrap() as u64;
       let jackpot_cut = (total_pool as u128)
           .checked_mul(JACKPOT_BPS as u128).unwrap()
           .checked_div(10_000).unwrap() as u64;
       let rake = treasury_cut + jackpot_cut; // Avoids separate rake calc + rounding

       // Direct lamport mutation (program-owned account)
       **pixel_state.to_account_info().try_borrow_mut_lamports()? -= rake;
       **treasury.to_account_info().try_borrow_mut_lamports()? += treasury_cut;
       **jackpot.to_account_info().try_borrow_mut_lamports()? += jackpot_cut;
   }
   ```
4. Zero-winner branch (SC-13):
   ```rust
   let winning_pool = pixel_state.color_pools[winning_color as usize];
   if winning_pool == 0 && total_pool > 0 {
       // No one bet on the winning color -- send net pool to treasury
       let net_pool = total_pool.checked_sub(treasury_cut + jackpot_cut)
           .ok_or(error!(PixelPredictError::ArithmeticOverflow))?;
       // Ensure we retain rent-exempt minimum
       let rent = Rent::get()?.minimum_balance(pixel_state.to_account_info().data_len());
       let available = pixel_state.to_account_info().lamports()
           .checked_sub(rent)
           .ok_or(error!(PixelPredictError::InsufficientFunds))?;
       let transfer_amount = net_pool.min(available);

       **pixel_state.to_account_info().try_borrow_mut_lamports()? -= transfer_amount;
       **treasury.to_account_info().try_borrow_mut_lamports()? += transfer_amount;
   }
   ```
5. Season completion (SC-14):
   ```rust
   season_state.current_pixel_index = season_state.current_pixel_index.checked_add(1)
       .ok_or(error!(PixelPredictError::ArithmeticOverflow))?;
   let total_pixels = (season_state.grid_width as u16) * (season_state.grid_height as u16);
   if season_state.current_pixel_index >= total_pixels {
       season_state.status = SeasonStatus::Completed;
       season_state.completed_at = Some(Clock::get()?.unix_timestamp);
   }
   ```

**claim_winnings instruction:**

Accounts struct `ClaimWinnings`:
- `pixel_state: Account<'info, PixelState>` with `mut`, seeds, bump
  - Constraint: `pixel_state.status == RoundStatus::Resolved @ PixelPredictError::RoundNotResolved`
- `bet_account: Account<'info, BetAccount>` with `mut`, seeds: `[b"bet", pixel_state.season_number.to_le_bytes(), pixel_state.pixel_index.to_le_bytes(), player.key()]`, bump = bet_account.bump
  - Constraint: `!bet_account.claimed @ PixelPredictError::AlreadyClaimed`
  - Constraint: `bet_account.player == player.key()` (authorization -- player can only claim their own)
- `player_season_stats: Account<'info, PlayerSeasonStats>` with `mut`, seeds: `[b"stats", pixel_state.season_number.to_le_bytes(), player.key()]`, bump = player_season_stats.bump
- `/// CHECK: player receives lamports` `player: Signer<'info>` with `mut`
- `system_program: Program<'info, System>`

Arguments: none

Handler logic:
1. Verify winner: `bet_account.color == pixel_state.winning_color.unwrap()` (error: NotWinner). Use unwrap safely because status == Resolved guarantees winning_color is Some.
2. Calculate payout using `crate::payout::calculate_winner_payout(bet_account.amount, pixel_state.color_pools[winning_color as usize], pixel_state.total_pool)`
3. Rent protection: calculate minimum rent for PixelState. Available = pixel_state.lamports() - rent_minimum. If payout > available, cap at available. This handles the edge case where multiple claims + rake drain the account.
4. Transfer via direct lamport mutation:
   ```rust
   **pixel_state.to_account_info().try_borrow_mut_lamports()? -= payout;
   **player.to_account_info().try_borrow_mut_lamports()? += payout;
   ```
5. Set `bet_account.claimed = true`
6. Increment `player_season_stats.correct_predictions += 1` (SC-15: updated at claim time, not resolve time)
