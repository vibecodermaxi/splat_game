Build the 10x10 CSS Grid canvas with live pixel rendering, active pixel animation, pixel tooltips, season progress indicator, and real-time data subscription via Helius WebSocket with polling fallback.

Purpose: The canvas is the hero of the game UI. Players must see the evolving artwork, identify the active pixel for betting, and receive live updates as the round progresses. The WebSocket subscription is the real-time data backbone for the entire frontend.

Output: A visible 10x10 canvas that renders resolved pixels with shade/warmth colors, pulses the active pixel with a marching-ants border, shows tooltips on tap, displays season progress, and stays synced with on-chain state via WebSocket.
</objective>

    1. Create `app/src/hooks/useSeasonData.ts` ("use client"):
       - On mount: fetch the current SeasonState account (derive PDA from seasonNumber in store, or try season 1 by default)
       - Decode with program.coder.accounts.decode("SeasonState", accountInfo.data)
       - Extract: seasonNumber, currentPixelIndex, gridWidth, gridHeight, status
       - Update Zustand store with setSeasonState(seasonNumber, currentPixelIndex)
       - Then fetch PixelState for ALL resolved pixels (0 to currentPixelIndex-1) plus the active pixel
       - Use program.account.pixelState.fetchMultiple() with all pixel PDAs for batch fetching
       - Decode each and call setPixelState for each pixel
       - Also fetch player's BetAccount for the active pixel (if wallet connected) and setPlayerBet
       - Return { loading: boolean, error: string | null }

    2. Create `app/src/hooks/useHeliusSocket.ts` ("use client"):
       - Params: pixelPDA (PublicKey | null), seasonPDA (PublicKey)
       - Uses useConnection() for the Connection object
       - Uses useAnchorProgram() for the program coder (to decode raw buffers)
       - On mount (when pixelPDA is set):
         a. Subscribe to pixelPDA via `connection.onAccountChange(pixelPDA, callback, "confirmed")`
         b. In callback: decode buffer with `program.coder.accounts.decode("PixelState", info.data)`
         c. Convert decoded data to PixelSnapshot (convert BN fields with .toNumber()), call setPixelState
         d. Check if status changed to "resolved" — if so, also re-fetch SeasonState to get new currentPixelIndex
       - Subscribe to seasonPDA similarly for SeasonState changes
       - Ping: setInterval(() => connection.getSlot(), WS_PING_INTERVAL_MS) to prevent Helius 10-min timeout
       - Track wsConnected state via store.setWsConnected(true) on first successful callback
       - On WebSocket error/close: set wsConnected(false), start polling fallback
       - Polling fallback: setInterval at POLLING_FALLBACK_INTERVAL_MS — fetches PixelState account via program.account.pixelState.fetch(pixelPDA), decodes, updates store
       - On tab visibility change (document.visibilitychange): when visible, immediately re-fetch and reset intervals (prevents stale data from browser throttling — Pitfall 5)
       - Cleanup: removeAccountChangeListener, clear all intervals on unmount
